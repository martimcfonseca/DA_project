Implements the Edmonds-Karp algorithm to compute the maximum flow.

The algorithm uses a breadth-first search (BFS) to find augmenting paths in the residual graph, updating capacities until no more paths exist from the source to the target.

Parameters
gPointer to the graph (flow network).
sourceSource node.
targetTarget node (sink).
Note
Time Complexity: O(V · E²), where V is the number of vertices and E is the number of edges. This is because each BFS takes O(E) and the number of augmenting paths in the worst case is O(V · E).

Processes the graph after the maximum flow computation and exports the assignment results.

This function iterates through the graph edges to identify assignments between reviewers and submissions (where flow > 0), organizes the data in maps for ordered output, writes the results to a CSV file, and reports any missing reviews per submission.

The output file contains:

  • SubmissionId, ReviewerId, Match
  • ReviewerId, SubmissionId, Match
  • Total number of assignments
  • Missing reviews per submission (if any)
Parameters
graphReference to the processed flow graph.
dataReference to the input data.
outputFileName of the output file where results will be written.
Note
Time Complexity: O(R × A + V + E), where R is the number of reviewers, A is the average number of assignments per reviewer, V is the number of vertices, and E is the number of edges in the graph.
